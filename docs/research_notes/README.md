# Bitácora técnica del proyecto

Esta carpeta conserva el contexto acumulado del proyecto para que las decisiones no queden solo en conversaciones.

## Documentos

- `01_estado_actual.md` — qué existe hoy, qué tan sólido es y qué falta.
- `02_decisiones_tecnicas.md` — decisiones tomadas y por qué.
- `03_plan_evaluacion_y_benchmark.md` — cómo evaluar el futuro modelo SFT Q+A cuando no existe benchmark externo alineado.
- `../article_methodology.md` — síntesis metodológica para redactar el artículo del dataset.

## Cómo usar esta carpeta

1. Antes de cambiar el pipeline, revisa `01_estado_actual.md`.
2. Si tomas una decisión nueva, agrega o actualiza una fila en `02_decisiones_tecnicas.md`.
3. Si cambian las reglas de evaluación, actualiza `03_plan_evaluacion_y_benchmark.md`.
